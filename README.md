# 특이점이 온다 — 독서 모임 대시보드

3명(규리·진영·유리)이 같은 URL로 접속해 모임 현황을 볼 수 있는 대시보드.

## 파일 구성
- `index.html` — 대시보드 본체 (싱글 파일, 의존성 없음)
- `chat.txt` — 카카오톡 단톡방 대화 내보내기 파일 (이걸 주기적으로 갱신하면 대시보드가 자동 반영)
- `README.md` — 이 파일

## 로컬에서 바로 보기
`index.html`을 더블클릭하면 브라우저에서 열립니다. 같은 폴더의 `chat.txt`를 자동으로 읽어옵니다. 또는 다른 txt 파일을 페이지에 드래그하면 즉시 분석됩니다.

## 카톡 대화 내보내기
- **PC 카카오톡:** 단톡방 우측 상단 ≡ → 대화 내보내기 → 텍스트만 → 저장
- **모바일 카톡:** 단톡방 메뉴 → 대화 내보내기 → 텍스트 메시지만 내보내기 → 파일로 저장 (메일/드라이브로 PC로 전송)

내보낸 `.txt`를 폴더의 `chat.txt`로 덮어쓰면 끝.

## 3명이 같은 URL로 보기 — GitHub Pages 배포 (10분, 무료)

### 1. GitHub 계정 만들기
https://github.com 에서 가입 (없다면)

### 2. 새 저장소 만들기
- 상단 `+` → **New repository**
- Repository name: `book-club-singularity` (아무 이름이나)
- **Public** 선택 (Private은 Pages 유료)
- **Add a README file** 체크
- **Create repository**

### 3. 파일 업로드
- 저장소 페이지에서 **Add file** → **Upload files**
- `index.html`, `chat.txt` 두 파일 드래그
- 아래 **Commit changes** 클릭

### 4. Pages 활성화
- 저장소 상단 **Settings** 탭
- 좌측 사이드바 **Pages**
- Source: **Deploy from a branch**
- Branch: **main**, 폴더: **/ (root)** → **Save**
- 1~2분 후 페이지 상단에 URL이 표시됨 (`https://<아이디>.github.io/book-club-singularity/`)

### 5. 3명에게 URL 공유
끝. 누구나 그 URL로 접속해 현황 확인 가능.

### 6. 데이터 업데이트 (주 1~2회)
- 카톡에서 대화 내보내기 → `chat.txt` 받음
- GitHub 저장소에서 `chat.txt` 클릭 → 연필 아이콘(편집)
- 받은 파일 내용을 통째로 붙여넣기 → **Commit changes**
- 1분 후 대시보드 반영

> 누가 업데이트 담당할지 정해두면 좋아요. (예: 매주 일요일 밤 한 명이)

## 검출 규칙
메시지 본문에 다음 패턴이 있어야 인식됩니다:
```
M/D 이름_N회차 미션완료
```
예: `5/15 규리_5회차 미션완료`, `5/18 유리_6회차 미션완료`

- 띄어쓰기 변형(`미션 완료`, `회 차`)은 허용
- 같은 사람의 같은 회차가 중복되면 첫 번째 것만 인정
- 회차 번호 기준으로 진도 판정 (메시지 안의 날짜는 정시/만회 판정에만 사용)

## 색상 의미
- 🟢 **정시 완료** — 그 회차의 예정일 또는 그 이전 날짜로 표기
